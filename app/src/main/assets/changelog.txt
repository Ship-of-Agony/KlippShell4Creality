v0.8.9.140626-rc

• Discrete Multi-Channel Notifications & Proactive OS Registration: Implemented two separate, user-configurable notification channels (NotificationChannel) to comply with modern Android TV guidelines. Segregated alerts into high-priority heads-up banners for critical printer events ("Fehler & Statusmeldungen") and standard background notifications ("Informationen & Meilensteine"). Channels are now registered proactively during activity startup (onCreate), making them instantly visible and controllable within the Android system settings without requiring a prior event. The POST_NOTIFICATIONS runtime permission was anchored in the manifest and seamlessly coupled with the onboarding privacy disclosure dialog.
• Type-Safe Layout Architecture & Reflection Elimination: Restructured the layout parameter assignment for the exit button (btnExitApp) within MainActivity.kt. A risky Java reflection hack (params.javaClass.getField) that manipulated private fields of the ConstraintLayout library was completely eliminated and replaced with compile-time checked, type-safe cast assignments to ConstraintLayout.LayoutParams to guarantee layout compatibility across future system updates.
• Comprehensive i18n Toast Synchronization & Localization: Cleaned up residual untranslated text fragments inside the video toggle logic (btnToggle) within WebViewActivity.kt. The toast alerts now dynamically load fully translated resource IDs (R.string.toast_loading_dashboard and R.string.toast_loading_livestream) using getString() instead of hardcoded strings, ensuring a flawless internationalization experience across all 7 targeting languages.
• Dynamic Color Synchronization & Design Integrity: Removed all hardcoded hex color values from the centered pill toasts and the OSD background configurations (both compact box and flexible banner layouts). Backgrounds and borders now pull their structural color values programmatically via ContextCompat.getColor() from adaptive resource IDs (R.color.pill_normal_inactive and R.color.pill_normal_inactive_text), guaranteeing a flawless visual integration across both light and dark display modes.
• Network Scan Timeout Calibration: Optimized the connection timeout within the automatic printer broadcast scanner (searchNetworkForPrinters) from a brittle 350ms to a highly stable 750ms. This calibration gracefully handles packet drops and latency spikes typical of weak Wi-Fi hardware antennas in budget Android TV sticks, drastically increasing the printer auto-discovery rate without causing any noticeable delay to the parallelized scanning task.
• Compiler Integrity & Syntax Clean-up: Resolved critical Kotlin compilation errors encountered during refactoring within WebViewActivity.kt. Eliminated accidental duplicate updateOsdTextForm method declarations, fixed broken brace nesting scopes that had hidden the fetchMoonrakerData method, corrected an illegal wildcard variable reference inside the log catch block, and restored misplaced state-tracking variables to ensure absolute build integrity.
